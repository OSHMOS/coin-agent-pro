# 🪙 Coin Agent 트레이딩 봇 규칙 (초단타 스캘핑 모드)

이 파일은 현재 봇이 따르고 있는 알고리즘 매매 규칙을 설명합니다.
현재 이 규칙은 `.env` 파일과 `bot.js`의 내부 로직에 의해 제어되고 있습니다.

## 1. ⚙️ 기본 설정
- **타겟 코인**: BTC(비트코인), ETH(이더리움), XRP(리플), SOL(솔라나), DOGE(도지코인)
- **감시 주기**: 30초 (30초마다 5개 코인의 가격과 지표를 계산)
- **1회 매수 금액**: 10,000원
- **하루 최대 거래 횟수**: 100회
- **재매수 쿨다운**: 6분 (같은 코인을 매수한 후 최소 6분은 지나야 다시 살 수 있음)

## 2. 🟢 매수 로직 (Buy)
다음 **두 가지 조건이 동시에 충족**될 때만 10,000원어치 시장가(최우선 매도호가) 매수.
1. **RSI(14) ≤ 45**: 시장이 단기적으로 과매도 구간(조금 떨어짐)에 진입했을 때
2. **현재가 < MA(20)**: 현재 가격이 최근 20개 체결가의 평균(이동평균선)보다 아래에 있을 때 (눌림목)

## 3. 🔴 매도 로직 (Sell)
보유 중인 코인에 대해 다음 **세 가지 중 하나라도 충족**되면 즉시 전량 매도.
1. **[익절] 수익률 ≥ +1.5%**: 샀던 가격보다 1.5% 오르면 미련 없이 매도하여 이득 챙김
2. **[손절] 수익률 ≤ -3.0%**: 샀던 가격보다 3.0% 떨어지면 더 큰 손실을 막기 위해 눈물을 머금고 손절 
3. **[과매수 청산] RSI(14) ≥ 65 AND 수익 중**: 아직 +1.5%에는 도달 못 했지만, 차트가 과열(RSI 65 돌파)되었고 현재 단돈 1원이라도 수익(+%) 중이라면 욕심내지 않고 던짐

## 4. 💀 자폭 프로토콜 (생존 함수)
- 만약 코인이 다 팔린 상태에서 통장(코인원)에 남은 원화(KRW)가 **500원 미만**이 되면, 최소 주문 금액을 맞출 수 없어 더 이상 매매가 불가능함.
- 이때 봇은 스스로 프로세스를 강제 종료(`process.exit`)하며 생을 마감합니다.
